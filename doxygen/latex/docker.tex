\doxysection{Docker Installation}
\hypertarget{docker}{}\label{docker}\index{Docker Installation@{Docker Installation}}
Power\+Blocks provides ready to go Docker packages allowing easy use across all platforms.

\DoxyHorRuler{0}
 \begin{DoxyNote}{Note}
These instructions have not been tested on mac\+OS. If you try them and they work, please confirm by submitting a comment or issue in the \href{https://github.com/rainbain/PowerBlocks/issues}{\texttt{ Power\+Blocks Issue Tracker}}. Once mac\+OS compatibility is verified, this note will be removed. 
\end{DoxyNote}
\hypertarget{docker_autotoc_md25}{}\doxysubsection{\texorpdfstring{Prerequisites}{Prerequisites}}\label{docker_autotoc_md25}
To begin, you will need an installation of Docker installed. The version of Docker depends on the host OS.\hypertarget{docker_autotoc_md26}{}\doxysubsubsection{\texorpdfstring{Linux}{Linux}}\label{docker_autotoc_md26}
For Linux you can install Docker Engine directly from \href{https://docs.docker.com/engine/}{\texttt{ here}}.\hypertarget{docker_autotoc_md27}{}\doxysubsubsection{\texorpdfstring{Windows \& Mac\+OS}{Windows \& Mac\+OS}}\label{docker_autotoc_md27}
For these systems, Docker Desktop is required and can be setup from \href{https://docs.docker.com/desktop/}{\texttt{ here}}.

\DoxyHorRuler{0}
\hypertarget{docker_autotoc_md29}{}\doxysubsection{\texorpdfstring{Start A Project}{Start A Project}}\label{docker_autotoc_md29}
\hypertarget{docker_autotoc_md30}{}\doxysubsubsection{\texorpdfstring{Create Project Folder}{Create Project Folder}}\label{docker_autotoc_md30}
To begin, a Power\+Blocks project using a Docker container, first locate the \href{https://github.com/rainbain/PowerBlocks/pkgs/container/powerblocks}{\texttt{ latest Power\+Block\textquotesingle{}s package}}.

Then, in a terminal, create a workspace. This can be the name of your project.\hypertarget{docker_autotoc_md31}{}\doxysubsubsubsection{\texorpdfstring{Example}{Example}}\label{docker_autotoc_md31}

\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ MyProject}
\DoxyCodeLine{cd\ MyProject}

\end{DoxyCode}
\hypertarget{docker_autotoc_md32}{}\doxysubsubsection{\texorpdfstring{Run Docker Container}{Run Docker Container}}\label{docker_autotoc_md32}
Now run the Docker container from inside your project folder. We will mount the project folder inside the container as {\ttfamily /workspace}.

Make sure to replace {\ttfamily \texorpdfstring{$<$}{<}VERSION\texorpdfstring{$>$}{>}} with your version of Power\+Blocks.\hypertarget{docker_autotoc_md33}{}\doxysubsubsubsection{\texorpdfstring{Linux \& mac\+OS}{Linux \& mac\+OS}}\label{docker_autotoc_md33}

\begin{DoxyCode}{0}
\DoxyCodeLine{docker\ run\ -\/-\/rm\ -\/it\ -\/v\ "{}\$PWD:/workspace"{}\ ghcr.io/rainbain/powerblocks:slim-\/<VERSION>}

\end{DoxyCode}
\hypertarget{docker_autotoc_md34}{}\doxysubsubsubsection{\texorpdfstring{Windows Power\+Shell}{Windows Power\+Shell}}\label{docker_autotoc_md34}

\begin{DoxyCode}{0}
\DoxyCodeLine{docker\ run\ -\/-\/rm\ -\/it\ -\/v\ "{}\$\{PWD\}:/workspace"{}\ ghcr.io/rainbain/powerblocks:slim-\/<VERSION>}

\end{DoxyCode}
\hypertarget{docker_autotoc_md35}{}\doxysubsubsection{\texorpdfstring{Create Project}{Create Project}}\label{docker_autotoc_md35}
To create the project, copy an example from the SDK into your current project folder.\hypertarget{docker_autotoc_md36}{}\doxysubsubsubsection{\texorpdfstring{Example}{Example}}\label{docker_autotoc_md36}

\begin{DoxyCode}{0}
\DoxyCodeLine{cp\ -\/r\ powerblocks/examples/HelloWorld/.\ /workspace/}

\end{DoxyCode}


At this point, a Power\+Block\textquotesingle{}s project is created within your project folder.\hypertarget{docker_autotoc_md37}{}\doxysubsubsection{\texorpdfstring{Build the Project}{Build the Project}}\label{docker_autotoc_md37}
Whole still operating inside the Docker container, you can now configure and build the project. \hypertarget{docker_autotoc_md38}{}\doxysubsubsubsection{\texorpdfstring{Build Commands}{Build Commands}}\label{docker_autotoc_md38}

\begin{DoxyCode}{0}
\DoxyCodeLine{cd\ /workspace}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cd\ build}
\DoxyCodeLine{cmake\ ..}
\DoxyCodeLine{ninja}

\end{DoxyCode}


You should now see a {\ttfamily Hello\+World.\+elf} inside the {\ttfamily build/} folder of your project.
